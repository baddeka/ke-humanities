RDF — Before & After
Initial (erroneous) LLM output vs. manually corrected Turtle triples. This comparison is the core methodological contribution of the project: it demonstrates where LLMs fail when generating knowledge graph data.
Format: All triples use Turtle (TTL) serialisation with standard Wikidata prefixes. Verified Q-IDs come from live SPARQL queries — none are assumed.

Gap 1 — P1066 (student of): Bernardino Luini

✗ Initial LLM output (errors) 
# Error 1: plain string instead of entity URI
wd:Q8537 wdt:P1066 "Leonardo da Vinci" .

# Error 4: reversed subject/object
wd:Q762 wdt:P1066 wd:Q8537 .
# → reads "Leonardo was student of Luini"
✓ Corrected triple 
# Luini (Q8537) student of Leonardo (Q762)
# Confidence: MEDIUM — stylistic evidence strong,
# documentary proof limited.
wd:Q8537 wdt:P1066 wd:Q762 .
  1. String literal instead of entity URI. The model wrote "Leonardo da Vinci" (a plain text string). In Wikidata RDF, property values that refer to other items must be entity URIs: wd:Q762.
  2. Reversed relationship direction. The model wrote wd:Q762 wdt:P1066 wd:Q8537, which means "Leonardo da Vinci was a student of Bernardino Luini" — the exact opposite of the intended fact. P1066 (student of) must have the student as subject and the teacher as object.

Gap 3 — P135 (movement): Boltraffio & Luini

✗ Initial LLM output (errors) 
# Error 2: wrong property ID (P136 = genre, not P135)
wd:Q8537 wdt:P136 wd:Q1404472 .

# Error 3: invented / hallucinated Q-ID
wd:Q442528 wdt:P135 wd:Q998877 .
# Q998877 does not exist in Wikidata

# Error 8: completely wrong Q-ID (Q11635 = theatre)
wd:Q280851 wdt:P135 wd:Q11635 .
✓ Corrected triples 
# Verified: Q1404472 = Italian Renaissance
#           Q1474884 = High Renaissance

wd:Q442528 wdt:P135 wd:Q1404472 .  # Boltraffio
wd:Q442528 wdt:P135 wd:Q1474884 .  # also High Renaissance

wd:Q8537   wdt:P135 wd:Q1404472 .  # Luini

wd:Q280851 wdt:P135 wd:Q1404472 .  # Piero di Cosimo
  1. Wrong property ID: P136 instead of P135. P136 is "genre" (used for literary/musical genres); P135 is "movement" (used for art movements). One digit difference, entirely different meaning. The model consistently confused these two.
  2. Hallucinated Q-ID. The model produced Q998877 for "Italian Renaissance". This Q-ID does not exist in Wikidata. The correct identifier, found via SPARQL lookup on the label "Italian Renaissance", is Q1404472.
  3. Semantically wrong Q-ID. For Piero di Cosimo, the model assigned Q11635, which resolves to "theatre" — a completely unrelated concept. This is a case of superficial pattern matching without entity disambiguation.

Gap 2 — P195 (collection): Baronci Altarpiece

✗ Initial LLM output (errors) 
# Error 5: wrong property — P131 = admin location
wd:Q1778000 wdt:P131 wd:Q1524 .

# Error 6: museum as string literal
wd:Q1778000 wdt:P195 "Pinacoteca Vaticana" .

# Error 9: missing prefix declarations,
# then using p:/ps:/pq: without declaring them
wd:Q1778000 p:P195 [
    ps:P195 wd:Q46699 ;
    pq:P518 wd:Q1493045
] .
# → parse error: prefixes p:, ps:, pq: undeclared
✓ Corrected triples 
# Prefixes declared at file top (see above).
# Full statement structure used for qualifiers.

wd:Q1778000 p:P195 [
    ps:P195 wd:Q46699 ;
    pq:P518 "God the Father with Angels"^^xsd:string
] .

wd:Q1778000 p:P195 [
    ps:P195 wd:Q511959 ;
    pq:P518 "Angel fragment"^^xsd:string
] .

# NOTE: Verify Q46699 (Pinacoteca Vaticana)
# and Q511959 (Pinacoteca Tosio Martinengo)
# against museum catalogues before submitting.
  1. Wrong property for museum: P131 vs P195. P131 ("located in administrative territorial entity") links an item to a city or region; P195 ("collection") links an artwork to the institution that holds it. The model confused geographic location with institutional holding.
  2. String literal for museum name. "Pinacoteca Vaticana" is a text string, not a Wikidata entity. The correct form is wd:Q46699 — the entity representing that museum.
  3. Missing prefix declarations causing parse errors. The model used p:, ps:, and pq: prefixes for full statement structure without declaring them at the top of the document. Any Turtle parser would reject this file.

Key lesson: LLMs produce syntactically plausible-looking RDF that fails on semantic grounds — wrong Q-IDs, inverted relationships, wrong property numbers. Every LLM-generated triple must be verified against the live knowledge base before use.
