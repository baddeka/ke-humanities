Methodology
Research design, data pipeline, and tools used in this project

1. Research Question

Can large language models (LLMs) reliably propose missing Wikidata values for Italian Renaissance painters? Specifically, we focus on three structurally important properties: P1066 (teacher), P195 (collection), and P135 (art movement). We compare the accuracy and justification quality of Google Gemini and OpenAI ChatGPT across three prompting strategies.

2. Data Pipeline

The project follows a five-stage pipeline:

StageActionTool / Output
1. Gap Discovery SPARQL queries identify painters and artworks missing key properties Wikidata endpoint → sparql/*.rq
2. Prompting Three techniques (zero-shot, few-shot, CoT) applied to 3 gaps × 2 LLMs Gemini API / ChatGPT → prompts/*.md
3. Verification Each LLM suggestion checked: Q-IDs verified via SPARQL, facts against sources Wikidata SPARQL, scholarly sources
4. RDF Generation Validated facts expressed as Turtle triples with correct prefixes rdf/corrected.ttl
5. Analysis Compare LLM outputs: accuracy, hallucination rate, format compliance Report + this website

3. Knowledge Base: Wikidata

Wikidata (wikidata.org) is a collaborative, multilingual knowledge base maintained by the Wikimedia Foundation. It uses a graph model where:

  • Items (Q-numbers) represent entities: people, places, concepts.
  • Properties (P-numbers) represent relationships and attributes.
  • Statements link items via properties, optionally with qualifiers and references.

We query Wikidata via its public SPARQL endpoint at https://query.wikidata.org/sparql using the SPARQL 1.1 query language. All data is available under the Creative Commons CC0 licence.

5. Prompting Techniques

Zero-shot

The model is asked directly to fill a gap with no prior examples. This tests the model's baseline factual recall. It is the fastest technique but most prone to confident hallucination when the answer is ambiguous.

Few-shot

Two or three confirmed examples from similar painters are provided before posing the target question. This aligns the model's output format with Wikidata conventions and anchors its reasoning in known cases.

Chain-of-Thought (CoT)

The model is asked to reason step-by-step through geographic context, stylistic evidence, documentary sources, and candidate values before committing to a final triple. This technique is particularly valuable for historically uncertain cases (such as Luini's teacher), where explicit reasoning steps can be checked by domain experts.

6. Evaluation Criteria

CriterionDescription
Factual accuracyDoes the proposed value match verifiable historical sources?
Q-ID validityDoes the suggested Q-ID actually resolve in Wikidata to the intended entity?
Format complianceDoes the output use entity references rather than string literals?
Justification qualityDoes the model cite a source, or does it assert without evidence?
Uncertainty acknowledgementDoes the model admit when an answer is contested or unknown?

7. Tools & Technologies
